# 📘 Mini Blog API – Step-by-Step Execution Plan (No Code)
---

# 🧱 Phase 1: Project Setup

### Step 1:

* Ek new Node.js project initialize karo
* Required packages install karo:

  * server framework
  * database connector
  * authentication tools
  * validation tools

### Step 2:

* Folder structure create karo:

```
mini-blog-api/
│
├── src/
│   ├── config/
│   │   └── db.js                  # Database connection
│   │
│   ├── models/
│   │   ├── user.model.js          # User schema (name, email, password)
│   │   └── post.model.js          # Post schema (title, content, tags, authorId)
│   │
│   ├── controllers/
│   │   ├── auth.controller.js     # Register & Login logic
│   │   └── post.controller.js     # CRUD + Search/Sort/Pagination
│   │
│   ├── routes/
│   │   ├── auth.routes.js         # /api/auth/register, /api/auth/login
│   │   └── post.routes.js         # /api/posts (saare routes)
│   │
│   ├── middleware/
│       ├── auth.middleware.js     # JWT token verify karo
│       ├── ownership.middleware.js # Sirf apni post edit ho
│       └── error.middleware.js    # Standard error format
│    
│
├── .env                           # JWT_SECRET, DB_URL, PORT
├── .gitignore
├── package.json
└── server.js                      # Entry point (server start)

```

👉 Reason: Separation of concerns (clean architecture)

# 🗄️ Phase 2: Database Setup

### Step 3:

* Database connect karo (MongoDB)

### Step 4:

* `.env` file use karo:

  * database URL
  * secret keys

👉 Reason: Security + flexibility

---

# 👤 Phase 3: User System (Auth Base)

### Step 5: User Schema Design

Define:

* name
* email (unique)
* password
* timestamps

👉 Focus:

* Email duplication avoid karna
* Password plain text nahi hona chahiye

---

### Step 6: Register Flow

Flow samajh:

1. User data receive karo
2. Check karo email already exist hai ya nahi
3. Password hash karo
4. User save karo
5. Clean response bhejo

👉 Important:

* Password kabhi response me mat bhejna

---

### Step 7: Login Flow

Flow:

1. Email check karo
2. Password compare karo
3. Agar correct → token generate karo
4. Token client ko bhejo

👉 Token = identity proof

---

# 🔐 Phase 4: Authentication Middleware

### Step 8: Auth Middleware

Flow:

1. Header se token lo
2. Token verify karo
3. User info extract karo
4. Request me attach karo

👉 Reason:

* Har protected route me user identify karna

---

# 📝 Phase 5: Post System

### Step 9: Post Schema

Define:

* title
* content
* tags
* authorId (user reference)

👉 Relation:
User → Posts (1 to many)

---

### Step 10: Create Post

Flow:

1. Auth middleware se user mil jayega
2. User id ko authorId me store karo
3. Post save karo

👉 Rule:

* Sirf logged-in user post bana sakta hai

---

# 🔒 Phase 6: Ownership Middleware

### Step 11: Ownership Check

Flow:

1. Post find karo by ID
2. Compare:

   * post.authorId === loggedInUser.id
3. Match nahi hua → reject

👉 Reason:

* Security (dusra user edit/delete na kare)

---

# 📖 Phase 7: Read APIs

Yaha sabse important logic aata hai:

---

## 🔍 Search Logic

* Query parameter lo (`search`)
* Title + Content me match karo
* Partial matching use karo

👉 Example thinking:
"user 'node' search kare → 'Node Guide' bhi mile"

---

## ↕️ Sorting Logic

* Field select karo:

  * createdAt
  * title
* Order apply karo:

  * asc / desc

👉 Dynamic sorting banana hai

---

## 📄 Pagination Logic

Use:

```
skip = (page - 1) × limit
```

Flow:

1. Page lo
2. Limit lo
3. Skip calculate karo
4. Limited data return karo

👉 Reason:

* Performance optimization

---

### Step 13: Get Single Post
